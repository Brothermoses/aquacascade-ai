# AquaCascade AI

**Texas A&M Mays AI Venture Velocity Challenge 2026 — Stage 2 (Moise Tchivwila)**

A calibrated decision-support tool that helps a water system or a state
drinking-water program decide **which lead-status-unknown service lines
to investigate first** under EPA's Lead and Copper Rule Improvements
(LCRI). It is honestly scoped: it improves triage for systems and
programs **using their own historical SDWIS data**; it is not a
nationwide cold-start oracle. The honest scope statement is in the
operational system's footer and on every report.

## Honest scope (stated up-front, not in a footnote)

- The triage model is **system-level**, not per-pipe. A line's priority
  inherits its system's model rank.
- Within-distribution it is calibrated (ECE 0.024, Brier 0.079, ROC 0.71,
  top-decile captures ≈ 33 % of lead-rich systems). Under group-by-state
  CV it falls to **ROC 0.57** — it does NOT generalize cold to states
  it has never seen. That limitation is on its own slide and stated in
  the app footer.
- All cost / horizon / service-life figures are explicit engineering
  assumptions — never presented as measured savings.
- The operational system's nationwide architecture (multi-tenant
  scoping, atomic concurrency, national rollups) is verified at
  single-node scale on SQLite. A PostgreSQL / horizontal-scale /
  security-review production deployment is code-ready but not
  provisioned or verified.
- **No fabricated traction.** An earlier draft generated by another
  tool included invented interviews, signed pilots and a pipeline
  dollar figure; that material was disowned, quarantined locally, and
  is NOT included in this repository.

## Reproducibility

```bash
# operational system (local, offline)
cd 07_Tool
python -m aquacascade_system.app          # http://127.0.0.1:5000

# run the full automated test suite
python aquacascade_system/tests/run_tests.py

# reproduce a specific experiment log's numbers
cd ../05_Modeling
python signature_pipeline.py
python sig_depth_ab.py
python triage_production.py
```

EPA data is not in this repository (too large; freely re-downloadable
from EPA SDWIS / ECHO — see `01_Raw_Data/README.md`).

Python 3.12+ is supported. Dependencies: `flask`, `pandas`, `numpy`,
`scikit-learn`, `geopandas`, `matplotlib`, `python-docx`, `openpyxl`.
