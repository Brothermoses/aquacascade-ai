AquaCascade — Service-Line Investigation Register (local web app, v1)

RUN
  python 07_Tool/aquacascade_register/app.py
  then open  http://127.0.0.1:5000   (Ctrl+C to stop)

REQUIREMENTS
  Python 3, with: flask, pandas, openpyxl  (already installed in this env)

WHAT IT DOES (v1, decided in Report C)
  1. Import a utility's own service-line list (CSV/XLSX). Only a line-ID
     column is required; include a 'pwsid' column to pull in the model's
     system-level priority for ordering the worklist.
  2. Record each line investigation in an EPA-LCRR-compliant schema
     (system-owned + customer-owned material, basis, method, date, install
     year, disadvantaged-community flag, notes/photo). The overall LCRR
     classification is computed automatically.
  3. Export three files:
       /export/register.csv        full working register (all fields)
       /export/sdwis_rollup.csv    per-PWSID counts in the 4 SDWIS
                                    categories — the state submission
       /export/model_feedback.csv  predicted (model, system-level) vs
                                    confirmed (field) — retrains the model

DEMO
  Import  SAMPLE_service_lines_DEMO.csv  (synthetic, for testing only —
  real per-line lists must come from the utility; line-level data is not
  public). Its PWSIDs are real top-ranked systems so the model-priority
  join is visible.

DATA
  Local SQLite at  07_Tool/aquacascade_register/data/register.db
  No cloud, no network, no PII beyond what the utility imports.

REHABILITATION PLAN  (/rehab)
  After lines are confirmed, the Rehabilitation plan page turns the
  confirmed Lead / Galvanized-Requiring-Replacement lines into a
  multi-year replacement schedule: minimizes lead-exposure-years,
  front-loads disadvantaged-community lines (LCRI), within an annual
  budget and compliance horizon. It reports deadline feasibility (and
  the minimum budget to comply if not), an equity report, an honest
  comparison vs. oldest-first / as-found baselines, and exports the
  year-by-year plan (/export/rehab_plan.csv). Engine: rehab.py
  (standalone, self-testable: python rehab.py). Clustering deferred to v2.

HONEST SCOPE
  Prediction is SYSTEM-level, not per-line. The triage-model edge and the
  rehab ordering edge over naive are modest by design — the value is
  compliant capture, LCRI deadline feasibility, and equity scheduling,
  not a flashy savings number. Cost figures are explicit assumptions.
  Within-utility line-level prioritization and mobilization clustering
  are later phases.
